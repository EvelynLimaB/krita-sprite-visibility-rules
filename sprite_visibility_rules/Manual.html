Sprite Visibility Rules

This docker links the visibility of layers used for game sprites, expression parts, outfits, effects, and alternate states.

Rule types

  • Exclusive set: showing one member hides the other members. It can optionally require a fallback so at least one remains visible.
  • Linked set: showing or hiding one member applies the same state to all members.
  • Inverse pair: exactly two members always have opposite visibility.

Creating a rule

  1. Select two or more layers in Krita's Layers docker.
  2. Open Settings → Dockers → Sprite Visibility Rules.
  3. Choose Add from selected layers, configure the rule, and press OK.
  4. Save the .kra document. Rules are embedded in that document.

Important behavior

The plugin checks participating layer visibility at a configurable interval. Normal eye-icon clicks, shortcuts, compositions, and other scripts are detected. If rules conflict and form a cycle, the plugin refuses to flicker the layers and reports a warning.

Deleting and recreating a layer gives it a new internal ID. Select the replacement layers and use Rebind.
